**Experiment – 02**
**Build a Local Area Network Using Packet Tracer**

**Aim**
To build a LAN using a switch, configure IP addressing in the 192.168.1.0/24 network, and verify connectivity using ping and tracert commands in Cisco Packet Tracer.

**Software Required**
Cisco Packet Tracer 8.2.1

**Devices Required**

* 4 PCs
* 1 Cisco Switch (2960)
* Copper straight-through cables

**Procedure**

1. Open Cisco Packet Tracer and create a new file.
2. Place 4 PCs (PC0, PC1, PC2, PC3) in the workspace.
3. Place one Cisco 2960 switch.
4. Connect each PC to the switch using copper straight-through cables.

   * PC0 FastEthernet0 → Switch FastEthernet0/1
   * PC1 FastEthernet0 → Switch FastEthernet0/2
   * PC2 FastEthernet0 → Switch FastEthernet0/3
   * PC3 FastEthernet0 → Switch FastEthernet0/4
5. Wait for green lights to confirm physical connectivity.

**IP Configuration (192.168.1.0/24)**
Configure IP from Desktop → IP Configuration in each PC.

* PC0: 192.168.1.1
* PC1: 192.168.1.2
* PC2: 192.168.1.3
* PC3: 192.168.1.4
  Subnet Mask for all: 255.255.255.0

**Verification**

1. Open Command Prompt in any PC and type:
   `ipconfig` to verify IP address.

2. Test connectivity using ping:
   `ping 192.168.1.2`
   `ping 192.168.1.3`
   `ping 192.168.1.4`

3. Trace route using:
   `tracert 192.168.1.4`

**Expected Output**
Ping replies received successfully.
Traceroute shows only one hop because all PCs are connected to the same switch.

**Result**
Successfully built a LAN using a switch, configured IP addresses in the 192.168.1.0/24 network, and verified connectivity using ping and tracert in Cisco Packet Tracer.
